Extraction (Collecte API & Scraping) Actif

Acquisition et collecte de données brutes depuis de multiples sources :

  • Fichiers Locaux : Extraction des catalogues initiaux Netflix, Disney+ et Amazon Prime au format CSV depuis Kaggle.
  • API The Movie Database (TMDB) : Appels RESTful pour enrichir les films avec le budget, les revenus globaux, les genres, la durée et la note moyenne.
  • Web Scraping (Letterboxd) : Parsing du code HTML (BeautifulSoup et JSON-LD) pour extraire la note des cinéphiles et l'indicateur de popularité communautaire (nombre de "fans").

2

Transformation & Fuzzy Matching Actif

Nettoyage profond et résolution d'entités pour unifier les données hétérogènes :

  • Pré-traitement : Standardisation des dates au format ISO (YYYY-MM-DD), imputation des valeurs manquantes et homogénéisation des champs textuels (acteurs, réalisateurs, pays).
  • Fuzzy Matching : Croisement inter-plateformes basé sur la distance de Levenshtein (thefuzz). Deux films de la même année ayant une similarité de 90% ou plus sont fusionnés en une entité unique (ex: "Spider-Man" et "Spiderman").
  • Calculs Analytiques : Génération de nouvelles métriques à la volée telles que la rentabilité (Revenus / Budget) et le bénéfice net.

3

Injection Relationnelle (Load) Actif

Structuration et persistance des données nettoyées pour des requêtes analytiques :

  • Modélisation en Étoile : Organisation des données autour d'une table de faits centrale (Fact_Films) interconnectée à des tables de dimensions (Dim_Genres, Dim_Plateformes).
  • Mappage ORM (SQLAlchemy) : Utilisation d'un ORM pour prévenir les injections SQL et gérer efficacement les relations de type Many-to-Many (Lien_Film_Genre, Lien_Film_Plateforme).
  • Stockage local : Injection par lots (batch insert) dans une base de données SQLite 3 optimisée pour des lectures très rapides par le Dashboard Flask.
